# Introduction about Programming Language Paradigms :-

### 1. What is a Programming Language?

* A programming language is a formal set of instructions used to communicate with a computer to perform specific tasks.

###### It allows developers to:

* Write programs
* Solve problems
* Develop software & applications

### 2. Why Programming Languages are Important?

* To build software (apps, websites, systems)
* To automate tasks
* To control hardware
* To solve real-world problems efficiently

## Types of Programming Languages

1. ### Low-Level Languages

    * Close to hardware
    * Fast execution
    * Difficult to understand

   ###### Examples:

    * Machine Language (0s and 1s)
    * Assembly Language

2. ### High-Level Languages

    * Easy to read and write
    * Human-friendly syntax

   ###### Examples:

    * Java
    * Python
    * C++
    * JavaScript

### Based on Paradigm

| Type            | Description                | Example    |

## Procedural Programming

Procedural programming is a paradigm where a program is structured as a sequence of procedures or functions. Each procedure performs a specific task, and the focus is on how to perform tasks step by step.

###### Key Features:

1. Top-Down Approach:

    * Program is designed from main function to sub-functions.

2. Uses Procedures/Functions:

    * Code is divided into smaller reusable blocks called procedures or functions.

3. Separation of Data and Functions:

    * Data is stored separately, and functions operate on data.

4. Control Structures:

    * Uses loops, conditionals, and sequential execution.

5. Reusability:

    * Functions can be reused but have limited flexibility compared to objects.

###### Advantages:

* Simple to implement for small programs.
* Easy to follow logic for step-by-step execution.

###### Disadvantages:

* Poor modularity for large programs.
* Difficult to maintain and scale.
* Less secure because data is exposed and can be modified by any function.

###### Examples of Procedural Languages:

* C, Pascal, Fortran.

#### Sample Conceptual Example in C (Procedural Style):

```c
#include <stdio.h>

int add(int a, int b) {
    return a + b;
}

int main() {
    int sum = add(5, 10);
    printf("Sum is %d\n", sum);
    return 0;
}
```

## Object-Oriented Programming

Object-oriented programming is a paradigm where programs are structured around objects rather than functions. Each object encapsulates data (attributes) and behaviour (methods).

###### Key Features:

1. Encapsulation:

    * Data and methods are bundled together, restricting direct access to internal state.

2. Inheritance:

    * Objects can inherit properties and behaviours from other objects.

3. Polymorphism:

    * Same method can behave differently in different objects.

4. Abstraction:

    * Hides implementation details and exposes only necessary features.

5. Modularity:

    * Programs are divided into classes and objects, making them more manageable.

###### Advantages:

* Promotes reusability of code via classes and objects.
* Easier to maintain and scale large programs.
* Better security and data protection.
* Closer to real-world modelling.

###### Disadvantages:

* More complex than procedural programming.
* Can be slower due to object overhead.

###### Examples of Object-Oriented Languages:

* Java, C++, Python.

#### Sample Conceptual Example in Java (OOP Style):

```java
class Calculator {
    int add(int a, int b) {
        return a + b;
    }
}

public class Main {
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        int sum = calc.add(5, 10);
        System.out.println("Sum is " + sum);
    }
}
```

### Programming languages need translators to convert code into machine language:

1. Compiler

    * Converts whole program at once
    * Example: Java (javac)

2. Interpreter

    * Converts line-by-line
    * Example: Python

3. Assembler

    * Converts assembly → machine code
